# Tech Debt Scan — 技术债穷举扫描

## 设计哲学

来自美团实践：**"核心开发圈定高危排查边界，AI 做穷举扫描"**。人类判断"扫哪里"和"哪个重要"，AI 负责"一个不漏地看"。

## 核心原则

1. **定向穷举**：在指定范围内，对每个文件全量检查，不遗漏
2. **人判优先级**：AI 给初步分级建议，P0/P1 最终由人确认
3. **可追踪**：债务清单可标记完成，下次扫描自动 diff

## 执行流程

### Phase 1: 确定扫描范围

向用户确认：
1. **扫描范围**：指定模块/包（推荐）或全项目
2. **债务类型**（默认全选）：
   - 分层违反 / 循环依赖 / 重复代码 / 缺失错误处理
   - 硬编码配置 / N+1查询风险 / 大函数大类 / 命名不规范 / 废弃代码

### Phase 2: AI 穷举扫描

对每个文件逐项检查：

| 债务类型 | 检测方法 |
|---------|---------|
| **分层违反** | 读 CLAUDE.md 分层规则 → 检查每个文件的 import 方向 |
| **循环依赖** | 构建依赖图 → 检测 A→B→A 循环 |
| **重复代码** | 提取方法体 → 归一化变量名 → AST 结构相似度比较（阈值70%） |
| **缺失错误处理** | 搜索空 catch 块、外部调用无 try-catch |
| **硬编码配置** | 搜索魔法数字、硬编码 URL/路径/超时值 |
| **N+1查询** | 搜索循环体内的 DB/RPC/HTTP 调用 |
| **大函数/大类** | 统计方法行数(>50标记)、文件行数(>500标记) |
| **命名不规范** | 检测拼音、过度缩写、类型信息内嵌、单字母变量 |
| **废弃代码** | 注释掉的代码块(3+行)、@Deprecated 仍有调用、无引用公开方法 |

### Phase 3: 自动分级

| 级别 | 判定规则 |
|------|---------|
| **P0** | 安全漏洞、数据丢失风险、死循环/死锁 |
| **P1** | 分层违反、循环依赖、N+1查询、重复代码(≥3处)、空catch吞异常 |
| **P2** | 大函数、大类、命名不规范、硬编码非安全配置、废弃代码 |

⚠️ P0/P1 最终由人确认，AI 分级仅为建议。

### Phase 4: 输出债务清单

写入 `.claude/tech-debt.md`，按 P0/P1/P2 组织，每项包含：描述、位置、类型、状态。

终端输出摘要：扫描范围、发现总数、各级别分布。

### Phase 5: 人类确认

1. 核心开发逐项确认/调整 P0/P1 分级
2. 为 P0/P1 匹配"顺带业务需求"
3. 更新状态和顺带需求列

## 增量扫描

后续运行默认仅扫描变更文件（git diff），`--full` 标志触发全量扫描。自动与上次报告 diff：标记新增/已修复。
